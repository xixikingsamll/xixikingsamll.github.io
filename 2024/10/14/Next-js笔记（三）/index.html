2. getStaticProps 静态生成

getStaticProps是一个 Next.js 提供的函数，用来在构建应用时获取数据，也就是说这是在服务端运行的函数，不会在浏览器上运行，这样的话能大大提高页面的加载速度，对 SEO 非常友好，主要适用的场景是：

  • 博客文章，商品列表等一些静态内容页面

提问：假如在vue中，`getStaticProps` 运行的时期应该最接近哪个生命周期？

答：getStaticProps 的功能可以类比为在组件初始化时获取数据并将其传递给组件进行渲染。在 Vue 中最接近的生命周期钩子是 beforeCreate 或 created，但它本质上与服务端渲染 (SSR) 或静态生成的方式更接近。

3. getServerSideProps 服务端渲染

getServerSideProps是 Next.js 中用于服务端渲染的一种数据获取方式顾名思义，每次请求页面的时候都会执行一次，不同于getStaticProps只在构建的时候执行。由于getServerSideProps可以保证在每次请求时从服务端获取最新的数据，并在页面加载时完成服务端渲染。它非常适合用于动态内容更新的页面，例如用户个性化页面、显示最新数据的页面等。

getServerSideProps的传参相比于getStaticProps的更加丰富，包含以下常用的属性：

  • params: 包含动态路由参数。

  • req: Node.js 请求对象。

  • res: Node.js 响应对象。

  • query: 包含 URL 查询参数。

  • locale: 当前语言环境。

  • resolvedUrl: 完整的请求 URL。

这其实也体现了其动态获取数据的能力

3.3 getServerSideProps 和 getStaticPropsde 对比

特性 getStaticProps getServerSideProps
数据获取时机 构建时 每次请求时
执行方式 静态生成 服务端渲染
适用场景 静态页面、数据变化不频繁 动态数据、实时更新的数据
性能 页面加载速度快，不消耗服务器资源 每次请求时获取数据，可能影响加载速度
SEO 友好 是 是
更新机制 可选增量静态生成（ISR） 每次请求时都会更新
